fix(swap): add Solana/Noble limit-swap destination validators + 1inch affiliate fee display (THOR-04/AGG-05) — `limitSwapDestinationValidators` (`limitSwapMemo.ts`) was missing `Chain.Solana` and `Chain.Noble`, even though both chains are already valid regular-swap destinations (`NativeSwapChain.ts`), so a limit swap targeting either failed closed with "chain not supported". Fixing the destination validator alone wasn't enough: the earlier `getSupportedThorchainAssetChain` prefix-resolution step also rejected `SOL.*`/`NOBLE.*` assets because it reused the LP-scoped `lpChainMap` (`chains/cosmos/thor/lp`, which never got Solana/Noble entries since neither supports THORChain LP) instead of the swap-scoped `nativeSwapChainIds` map that already lists them — switched the resolution to the latter, a strict superset that also unblocks previously-THOR-only-mapped assets like `MAYA.*`/`ADA.*` behind the existing per-chain validator gate. Separately, 1inch general-swap quotes never populated `affiliateFee` on the returned `GeneralSwapQuote`, unlike Kyber and LI.FI, leaving the fee-transparency row blank for 1inch even though a real affiliate cut is taken; `getOneInchSwapQuote` now grosses the post-fee `dstAmount` back up (same bps-based calc `getKyberSwapAffiliateFee` uses) to compute and attach a display-only `affiliateFee`.
