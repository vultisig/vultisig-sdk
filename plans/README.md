## Plan 005 outcome note (2026-07-08, branch advisor/005-vault-free-swap-safety-checks)

Landed in `packages/sdk/src/tools/prep/swap.ts` (`prepareSwapTxFromKeys`), scoped down from the
plan's own "Current state" assumptions after root-causing both checks against the live types:

**Expiry ŌĆö native quotes only.** `quote.native.expiry` is a real absolute deadline sourced from the
THORChain/Maya quote API; enforced, mirroring core's own `assertQuoteNotExpired`
(`nativeSwapQuoteToSwapPayload.ts`). General quotes carry **no expiry field at all** in the core
`SwapQuote` type the vault-free helper receives ŌĆö the plan's "general uses expiresAt ms" assumption
doesn't hold. The `expiresAt` `SwapService.prepareSwapTx` checks (`SwapService.ts:131`) is an
artificial SDK-wrapper clock computed fresh at fetch time (`Date.now() + DEFAULT_QUOTE_EXPIRY_MS`,
`SwapService.ts:360-361`), never part of the quote object itself ŌĆö nothing meaningful to enforce
here for general quotes, and not a regression (general already had zero enforcement in the
vault-free path).

**AmountŌåöquote cross-check ŌĆö landable only where the quote confidently commits to a sell amount
independent of the caller's input.** `general.tx.transfer.amount` (UTXO/Cosmos deposit-channel) and
`general.tx.cowswap_order.sellAmount` (CoW's signed EIP-712 order) are both directly comparable and
enforced. Native quotes have **no committed-sell-amount field anywhere** (checked the type + the
live THORChain quote response shape) ŌĆö the plan's own Step 2 escape hatch fires exactly as
anticipated. `evm`/`solana` general-quote amounts are encoded inside opaque aggregator calldata ŌĆö
deferred rather than inventing a comparison that could false-reject a legit swap.

**Named residual, not silently accepted:** after this lands, an **EVM-general vault-free swap**
(arguably the single most common shape) still has **neither** amount-consistency **nor** expiry
enforcement in the vault-free path ŌĆö the calldata-amount-decode gap defers both. Not a regression
(nothing enforced it before either), but it's the real remaining exposure once native + transfer/
cowswap_order are covered. Follow-up: an EVM calldata amount-decoder (analogous to what would unlock
decode-based checks elsewhere) is the prerequisite to close it ŌĆö worth tracking as a named next step,
not "good enough as-is."

10 new tests (`swap.test.ts`), all green; 3 pre-existing tests' quote fixtures updated (bare mock ŌåÆ
real `{quote:{native:{expiry}}}` shape) to exercise the real branches rather than break on the new
guards ŌĆö same outcomes, updated shape only. Full `yarn workspace @vultisig/sdk test`: 144 passed / 29
failed test files, 1 failed test ŌĆö matches the PRE-EXISTING baseline exactly (verified via `git
